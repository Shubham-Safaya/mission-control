Ninety days into any identity project, someone senior asks the question that should have opened the kickoff: what does done look like?

Identity resolution has no done. Populations churn. People move, marry, die, share phones, abandon emails. A graph that was ninety-five percent accurate in January degrades by December without a single code change, because the world it describes moved.

So the honest framing is not a project with an end date. It is an operating capability with error budgets, like uptime. Merge precision above X. Split rate below Y. Deletion completeness at Z within the statutory window. Measured continuously against a maintained truth set, staffed like a system that pages someone.

Selling that framing to leadership is genuinely hard, because capabilities lack ribbon-cuttings. The alternative is the two-year cycle I keep watching: launch, decay quietly, rediscover the decay in an audit, relaunch under a new name with a new vendor.

Name the maintenance cost upfront. It is smaller than the relaunch.
